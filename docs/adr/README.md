# Architecture Decision Records (ADR)

This folder stores key technical decisions that affect architecture and operations.

## Naming

- `NNNN-short-title.md` (example: `0001-market-registry-as-single-source.md`)

## Status

Use one of:

- Proposed
- Accepted
- Superseded
- Deprecated

## Process

1. Create a new ADR from `0000-template.md`.
2. Link related code/doc PRs.
3. Never delete ADRs; supersede them with a new one.
