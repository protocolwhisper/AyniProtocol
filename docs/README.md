# Documentation

This project uses a Diataxis-style documentation structure:

- `tutorials/`: learning-oriented, step-by-step guides
- `how-to/`: task-oriented operational guides
- `reference/`: factual, API/contract/config reference
- `explanation/`: architecture and design rationale
- `adr/`: architecture decision records

## Start here

- Architecture overview: `explanation/architecture.md`
- Architecture decisions: `adr/README.md`

## Contribution rules

- Keep each page focused on one purpose.
- Prefer small docs linked together over one large file.
- Include Mermaid diagrams where system flow is important.
